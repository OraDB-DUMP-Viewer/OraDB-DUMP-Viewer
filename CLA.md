Pull Request を提出することにより、以下のすべての条項に同意したものとみなされます。

### 1. 定義

- **「プロジェクト」** とは、OraDB DUMP Viewer およびその関連ソフトウェアを指します。
- **「コントリビューション」** とは、Pull Request、パッチ、コード変更、ドキュメント変更、
  その他プロジェクトに提出されるすべての成果物を指します。
- **「プロジェクトオーナー」** とは、YANAI Taketo を指します。

あなたは、コントリビューションに関する **すべての著作権および知的財産権** を、
プロジェクトオーナーに **無条件かつ取消不能に譲渡** します。

これには以下が含まれます：
- 複製権、翻案権、頒布権、公衆送信権
- 二次的著作物の作成権
- 商用利用を含むあらゆる利用形態での使用権
- 第三者へのサブライセンス権

### 3. オリジナリティの保証

あなたは以下を表明し保証します：
- コントリビューションがあなた自身のオリジナル作品であること

プロジェクトオーナーは、コントリビューションを含むプロジェクトを、
任意のライセンス条件（商用ライセンスを含む）で配布する権利を有します。

## 署名方法

Pull Request を提出すると、CLA Assistant が自動的に署名を求めます。
PR のコメント欄で指示に従い、CLA に署名してください。

署名は GitHub アカウントに紐づけて記録されます。
一度署名すれば、以降の PR では再署名は不要です。
