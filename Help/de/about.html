Version prüfen

Wählen Sie Hilfe(&H) > Über(&A) aus dem Menü, um die aktuelle Version zu prüfen.

Der Über-Dialog vergleicht automatisch mit dem neuesten GitHub-Release und zeigt an, ob ein Update verfügbar ist.

  • Bei Verwendung der neuesten Version: Die Versionsnummer wird grün angezeigt
  • Wenn eine neuere Version verfügbar ist: Die Versionsnummer wird orange angezeigt

Automatische Aktualisierung

Wenn eine neue Version verfügbar ist, erscheint eine Aktualisierungsschaltfläche im Dialog.

  1. Klicken Sie im Über-Dialog auf die Schaltfläche Auf neueste Version aktualisieren.
  2. Wählen Sie im Bestätigungsdialog Ja.
  3. Der neueste MSI-Installer wird automatisch heruntergeladen (ein Fortschrittsbalken wird angezeigt).
  4. Nach dem Download wird die Anwendung geschlossen und der Installer startet automatisch.

Sie können auch auf den Link Release-Seite öffnen klicken, um die GitHub-Release-Seite in Ihrem Browser anzuzeigen.

Die Lizenzverifizierung umfasst eine lokale RSA-Signaturprüfung und eine Online-Validierung beim Start. Offline-Umgebungen können weiterhin nur die lokale Verifizierung nutzen.

Erneute Authentifizierung oder Änderungen des Lizenzschlüssels können über Hilfe(&H) > Lizenzaktivierung vorgenommen werden.

Fehlerberichte

Wenn Sie einen Fehler oder ein Problem finden, melden Sie es über Hilfe(&H) > Fehlerbericht(&R).

Folgende Informationen werden automatisch erfasst und transparent im Berichtsdialog angezeigt:

  • Anwendungsversion / DLL-Version
  • Betriebssystemversion / .NET-Version
  • Architektur / Bildschirmauflösung / DPI
  • Dump-Dateiinformationen (nur Größe und Format; Pfade werden nicht gesendet)

Berichte werden automatisch als GitHub Issues über einen Cloudflare Worker übermittelt. Es werden keine persönlichen Informationen (Benutzername, PC-Name, Dateipfade usw.) erfasst oder gesendet.

Wenn die Dump-Datei 50 MB oder kleiner ist, können Sie sie optional dem Bericht anhängen.
