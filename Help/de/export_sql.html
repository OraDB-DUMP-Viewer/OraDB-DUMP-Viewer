SQL-Export

Vorgehensweise

  1. Wählen Sie die zu exportierende Tabelle in der Tabellenliste aus.
  2. Klicken Sie auf die Schaltfläche SQL in der Symbolleiste, oder wählen Sie Objekt(&O) > Export > Skript(&S).
  3. Geben Sie den Zieldateipfad an.
  4. Der Export wird ausgeführt.

Unterstützte DBMS

INSERT-Anweisungen können für folgende DBMS generiert werden:

DBMSSyntaxunterschiede
OracleTO_DATE-Funktion, NUMBER-Typ unverändert
PostgreSQLCAST-Syntax, TIMESTAMP-Typ
MySQLBacktick-Bezeichner, DATETIME-Typ
SQL ServerKlammer-Bezeichner, DATETIME2-Typ

Optionen

Beim Klicken auf die SQL-Schaltfläche wird vor dem Export ein Dialog angezeigt. Folgende Optionen können konfiguriert werden:

  • DBMS-Typ — Auswahl aus Oracle / PostgreSQL / MySQL / SQL Server
  • DROP TABLE + CREATE TABLE Ausgabe (Standard: EIN) — Per Kontrollkästchen umschaltbar. Wenn aktiviert, werden DROP TABLE IF EXISTS und CREATE TABLE DDL vor den INSERT-Anweisungen ausgegeben. Löscht und erstellt die Tabelle neu, falls sie bereits in der Zieldatenbank existiert.
  • Datumsformat — Literalformat oder Funktionsaufruf

DROP TABLE Syntax nach DBMS

DBMSSyntax
Oracle / PostgreSQLDROP TABLE IF EXISTS "table_name" CASCADE;
MySQLDROP TABLE IF EXISTS `table_name`;
SQL ServerIF OBJECT_ID('table_name', 'U') IS NOT NULL DROP TABLE [table_name];
Hinweis: LOB-Daten (BLOB/CLOB) sind nicht im SQL-Export enthalten. Verwenden Sie die LOB-Dateiextraktion, um LOB-Daten zu exportieren.
