SQL Export

Steps

  1. Select the table you want to export in the table list.
  2. Click the SQL button on the toolbar, or select Object(&O) > Export > Script(&S).
  3. Specify the destination file path.
  4. The export runs.

Supported DBMS

INSERT statements can be generated for the following DBMS:

DBMSSyntax Differences
OracleTO_DATE function, NUMBER type as-is
PostgreSQLCAST syntax, TIMESTAMP type
MySQLBacktick identifiers, DATETIME type
SQL ServerBracket identifiers, DATETIME2 type

Options

A dialog is displayed before export when you click the SQL button. The following options can be configured:

  • DBMS Type — Choose from Oracle / PostgreSQL / MySQL / SQL Server
  • DROP TABLE + CREATE TABLE Output (default: ON) — Toggle via checkbox. When ON, outputs DROP TABLE IF EXISTS and CREATE TABLE DDL before INSERT statements. Drops and recreates the table if it already exists in the target database.
  • Date Format — Literal format or function call

DROP TABLE Syntax by DBMS

DBMSSyntax
Oracle / PostgreSQLDROP TABLE IF EXISTS "table_name" CASCADE;
MySQLDROP TABLE IF EXISTS `table_name`;
SQL ServerIF OBJECT_ID('table_name', 'U') IS NOT NULL DROP TABLE [table_name];
Note: LOB data (BLOB/CLOB) is not included in SQL exports. Use LOB File Extraction to export LOB data.
