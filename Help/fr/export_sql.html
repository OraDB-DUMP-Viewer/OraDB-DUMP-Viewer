Export SQL

Étapes

  1. Sélectionnez la table que vous souhaitez exporter dans la liste des tables.
  2. Cliquez sur le bouton SQL de la barre d'outils, ou sélectionnez Objet(&O) > Exporter > Script(&S).
  3. Spécifiez le chemin du fichier de destination.
  4. L'export s'exécute.

SGBD pris en charge

Les instructions INSERT peuvent être générées pour les SGBD suivants :

SGBDDifférences de syntaxe
OracleFonction TO_DATE, type NUMBER tel quel
PostgreSQLSyntaxe CAST, type TIMESTAMP
MySQLIdentifiants avec accents graves, type DATETIME
SQL ServerIdentifiants entre crochets, type DATETIME2

Options

Une boîte de dialogue s'affiche avant l'export lorsque vous cliquez sur le bouton SQL. Les options suivantes peuvent être configurées :

  • Type de SGBD — Choisir parmi Oracle / PostgreSQL / MySQL / SQL Server
  • Sortie DROP TABLE + CREATE TABLE (par défaut : activé) — Activable/désactivable via case à cocher. Lorsqu'activé, génère le DDL DROP TABLE IF EXISTS et CREATE TABLE avant les instructions INSERT. Supprime et recrée la table si elle existe déjà dans la base de données cible.
  • Format de date — Format littéral ou appel de fonction

Syntaxe DROP TABLE par SGBD

SGBDSyntaxe
Oracle / PostgreSQLDROP TABLE IF EXISTS "table_name" CASCADE;
MySQLDROP TABLE IF EXISTS `table_name`;
SQL ServerIF OBJECT_ID('table_name', 'U') IS NOT NULL DROP TABLE [table_name];
Remarque : Les données LOB (BLOB/CLOB) ne sont pas incluses dans les exports SQL. Utilisez l'Extraction de fichiers LOB pour exporter les données LOB.
