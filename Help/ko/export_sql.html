SQL 내보내기

절차

  1. 테이블 목록에서 내보낼 테이블을 선택합니다.
  2. 도구 모음의 SQL 버튼을 클릭하거나 개체(&O) > 내보내기 > 스크립트(&S)를 선택합니다.
  3. 대상 파일 경로를 지정합니다.
  4. 내보내기가 실행됩니다.

지원 DBMS

다음 DBMS에 대한 INSERT 문을 생성할 수 있습니다:

DBMS구문 차이
OracleTO_DATE 함수, NUMBER 유형 그대로
PostgreSQLCAST 구문, TIMESTAMP 유형
MySQL백틱 식별자, DATETIME 유형
SQL Server대괄호 식별자, DATETIME2 유형

옵션

SQL 버튼을 클릭하면 내보내기 전에 대화 상자가 표시됩니다. 다음 옵션을 설정할 수 있습니다:

  • DBMS 유형 — Oracle / PostgreSQL / MySQL / SQL Server 중 선택
  • DROP TABLE + CREATE TABLE 출력 (기본값: ON) — 체크박스로 전환 가능. ON일 때 INSERT 문 앞에 DROP TABLE IF EXISTS 및 CREATE TABLE DDL을 출력합니다. 대상 데이터베이스에 테이블이 이미 존재하는 경우 삭제 후 다시 생성합니다.
  • 날짜 형식 — 리터럴 형식 또는 함수 호출

DBMS별 DROP TABLE 구문

DBMS구문
Oracle / PostgreSQLDROP TABLE IF EXISTS "table_name" CASCADE;
MySQLDROP TABLE IF EXISTS `table_name`;
SQL ServerIF OBJECT_ID('table_name', 'U') IS NOT NULL DROP TABLE [table_name];
참고: LOB 데이터(BLOB/CLOB)는 SQL 내보내기에 포함되지 않습니다. LOB 데이터를 내보내려면 LOB 파일 추출을 사용하세요.
